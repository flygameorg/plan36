# PROC/003 — 虚拟对话页面

一个纯前端、无需联网的选项式对话页面。玩家与"003"（一个老系统程序员角色）
通过点击选项聊天，系统在后台累计隐藏的"好感度"，根据最终结果决定要不要
发放一串随机密钥。

## 文件说明

- `index.html` — 页面结构，直接双击打开即可运行（也可以放到任意静态服务器上）。
- `style.css` — 视觉样式，琥珀色 CRT 老终端风格。
- `dialogue.js` — 所有对话文本、好感度数值、结局判定、密钥生成逻辑都在这里。
- `README.md` — 本文件。

## 好感度与结局是怎么算的

打开 `dialogue.js`，找到顶部的 `CONFIG`：

```js
const CONFIG = {
  threshold_good: 15,   // 达到或超过 -> 好结局 + 发密钥
  threshold_ok: 8,      // 达到或超过（但不到 good）-> 中间结局，可重来
  archiveInstruction: "……",
  lineDelay: 550,
};
```

`ROUNDS` 数组里一共 6 轮对话，每轮 3 个选项，每个选项有一个 `value`
（好感度增减，目前是 +3 / +1 / -2），总分区间大约是 -12 ~ 18。
选完全部 6 轮后，系统按总分落在哪个区间决定结局：

- `>= threshold_good` → 好结局，显示密钥
- `>= threshold_ok`（但 `< threshold_good`） → 中间结局，可以点"重新连接"再试一次
- 其余 → 冷结局，同样可以重来

**好感度全程不会显示在界面上**，玩家只能通过 003 的语气变化去猜。

## 怎么改对话文本

直接编辑 `dialogue.js` 里 `ROUNDS` 数组对应轮次的 `intro` / `options` /
`response` 文本即可，改完保存刷新页面就生效，不需要改 HTML 或 CSS。

每个选项长这样：

```js
{
  tier: "good",       // 仅供你自己编辑时辨认，不影响运行
  value: 3,           // 好感度变化
  text: "选项按钮上显示的文字",
  response: [
    { tag: "003", text: "003 说的话，一行一条" },
  ],
},
```

## 怎么设置密钥格式

密钥由 `generateKey()` 函数生成，默认格式类似：

```
003-K7XQ-9F3M-P2VD-77TN
```

字符集特意去掉了容易混淆的 `0/O`、`1/I/L`。如果想改长度、分组方式或前缀，
直接改这个函数即可。

## 怎么设置"取档案"的说明文字

`CONFIG.archiveInstruction` 里写你希望玩家看到的实际取件方式（比如具体网址、
邮箱、线下地点、下一步谜题入口等），密钥生成后会跟着这段文字一起显示。

## 已知的技术限制（请知悉）

好感度数值目前完全是在浏览器本地的 JavaScript 变量里计算的，**没有任何
服务器验证**。也就是说：

- 理论上非常熟悉浏览器开发者工具的玩家，可以直接在 `dialogue.js`
  源码里看到每个选项的 `value`，从而"看穿"好感度规则。
- 密钥虽然是真随机生成（用了 `crypto.getRandomValues`），但生成逻辑本身
  同样在客户端，硬核玩家理论上可以直接调用 `generateKey()` 拿到一个有效
  格式的密钥，而不需要真的走完对话。

对绝大多数 ARG 参与者来说这已经足够"隐藏"了，但如果你需要防止真正想
作弊的人绕过流程，就需要把好感度判定和密钥生成放到后端服务器上，前端
只负责展示——这个纯前端版本做不到这一点，仅供说明。

## 本地运行

直接双击 `index.html` 用浏览器打开即可，全程不需要联网、不需要安装任何
东西。如果想部署到网上，把这三个文件（`index.html` / `style.css` /
`dialogue.js`）放在同一个目录上传到任意静态托管即可。
